# 微信小程序抓包指南 - 获取 code 和 openID

## 问题原因

**微信的 code 只能使用一次！** 当你用抓包工具抓到 code 时，微信小程序已经使用了这个 code，所以再用脚本运行就会收到 400 错误。

**解决方法：必须在小程序使用 code 之前就拦截它！**

---

## 方法 1: Charles 抓包（推荐）⭐

### 步骤 1: 设置 Charles

1. **安装 Charles**
   - 下载：https://www.charlesproxy.com/
   - 安装 SSL 证书（Proxy → SSL Proxying Settings → Install Charles Root Certificate）

2. **配置 SSL 代理**
   - Proxy → SSL Proxying Settings
   - 勾选 "Enable SSL Proxying"
   - 添加 Location：
     - Host: `gate-obt.nqf.qq.com`
     - Port: `443`

3. **配置手机/模拟器代理**
   - WiFi 设置 → 配置代理 → 手动
   - 服务器：你的电脑 IP
   - 端口：8888（Charles 默认端口）

### 步骤 2: 设置断点拦截

1. **启用断点**
   - Proxy → Breakpoint Settings
   - 勾选 "Enable Breakpoints"
   - 添加断点：
     - Host: `gate-obt.nqf.qq.com`
     - Path: `/prod/ws`

2. **或者右键设置断点**
   - 在 Charles 中找到之前的 WebSocket 连接记录
   - 右键 → Breakpoints

### 步骤 3: 获取完整 URL

1. **重新打开微信小程序**（QQ 经典农场）
2. **Charles 会暂停请求**，弹出断点窗口
3. 在断点窗口中查看 **Request URL**，类似：
   ```
   wss://gate-obt.nqf.qq.com/prod/ws?platform=wx&os=iOS&ver=1.6.0.11_20251224&code=0e1E4l2w3JpBs630u41w34hTAX1E4l2-&openID=xxxxxx
   ```
4. **复制完整 URL**（包括 code 和 openID）

### 步骤 4: 立即运行脚本

1. **马上运行测试**：
   ```bash
   node ws_test.js "wss://gate-obt.nqf.qq.com/prod/ws?platform=wx&os=iOS&ver=1.6.0.11_20251224&code=YOUR_CODE&openID=YOUR_OPENID"
   ```

2. **如果测试成功，运行完整脚本**：
   ```bash
   # 从 URL 中提取 code 和 openID
   node client.js --code YOUR_CODE --wx
   ```

3. **在 Charles 中**：
   - 点击 "Cancel Request"（取消小程序的请求）
   - 或者点击 "Execute"（让小程序的请求继续，但 code 已被使用）

## 方法 2: Fiddler 抓包

### 步骤 1: 设置 Fiddler

1. **安装 Fiddler Classic**
   - 下载：https://www.telerik.com/fiddler

2. **配置 HTTPS 解密**
   - Tools → Options → HTTPS
   - 勾选 "Decrypt HTTPS traffic"
   - 安装证书

3. **允许远程连接**
   - Tools → Options → Connections
   - 勾选 "Allow remote computers to connect"

### 步骤 2: 设置断点

1. **启用自动断点**
   - Rules → Automatic Breakpoints → Before Requests

2. **或者使用命令**
   - 在底部命令行输入：`bpu gate-obt.nqf.qq.com`

### 步骤 3: 获取完整 URL

1. **重新打开微信小程序**
2. **Fiddler 会暂停所有请求**
3. 找到 `gate-obt.nqf.qq.com` 的 WebSocket 请求
4. 点击该请求，在右侧查看 **Raw** 或 **WebSocket** 标签
5. 复制完整的 URL

### 步骤 4: 运行脚本

同方法 1 的步骤 4

## 方法 3: Reqable（iOS 推荐）

如果你使用 iOS 设备，推荐使用 **Reqable**：

1. **安装 Reqable**
   - App Store 搜索 "Reqable"
   - 或访问：https://reqable.com/

2. **配置证书和代理**
   - 按照 App 内指引安装证书
   - 启用 HTTPS 抓包

3. **设置断点**
   - 在 Reqable 中找到 WebSocket 连接
   - 长按 → 添加断点

4. **获取 URL 并运行脚本**
   - 同方法 1

## 方法 4: 微信开发者工具（最简单，但可能缺少 openID）

### 获取 code

```javascript
wx.login({
  success: res => console.log('Code:', res.code)
});
```

### 问题

这个方法获取的 code 是全新的，没有被使用过。**但是可能没有 openID！**

如果服务器要求必须提供 openID，你还是需要用抓包工具获取完整的连接 URL。

---

## 重要提示

### ⚠️ 时间很关键！

1. **code 有效期约 5 分钟**
2. **code 只能使用一次**
3. **必须在小程序使用 code 之前拦截**
4. **拦截后立即运行脚本**

### ✅ 成功的标志

如果看到以下输出，说明连接成功：

```
[WS] connect url = wss://gate-obt.nqf.qq.com/...
========== 登录成功 ==========
  GID:    1234567890
  昵称:   你的昵称
  等级:   XX
  金币:   XXXX
===============================
```

### ❌ 失败的原因

- **400 错误**：code 已被使用或过期
- **missing openid parameter**：缺少 openID 参数

## 测试脚本

使用完整 URL 测试（推荐）：

```bash
# 直接用完整 URL 测试
node ws_test.js "wss://gate-obt.nqf.qq.com/prod/ws?platform=wx&os=iOS&ver=1.6.0.11_20251224&code=YOUR_CODE&openID=YOUR_OPENID"
```

如果连接成功会显示：
```
OPEN
```
